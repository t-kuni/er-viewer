# リサーチ依頼: アプリケーションの正式名称候補の提案

## 何をリサーチするか？

アプリケーションの正式名称の候補を提案してください。以下の要件を満たす名称案を複数提示してください。

### 依頼者から指示された要件

* 類似のツールやサービスと被らない固有名詞
* ER図のviewerであることが分かりやすい固有名詞
* 長すぎない。１単語か２単語程度が望ましい
* 造語でもOK

## プロジェクトの概要

このプロジェクトは、MySQLデータベースからER図をリバースエンジニアリングし、ブラウザ上で可視化・編集できるWebアプリケーションです。

### 主な機能

1. **リバースエンジニアリング**
   - MySQLデータベースからER図を自動生成
   - 増分リバース対応（既存の配置情報を維持）
   - DDL情報の保持

2. **可視化・編集機能**
   - インタラクティブなER図表示
   - エンティティのドラッグ&ドロップ配置
   - ズーム・パン操作
   - リレーション線の表示（直角ポリライン）
   - ホバー時のハイライト表示

3. **詳細情報表示**
   - エンティティクリックでDDL表示
   - サイドバーでの詳細情報表示
   - プライマリキー・外部キーの視覚的区別

4. **注釈機能**
   - 矩形描画（エンティティのグループ化用）
   - テキスト追加（補足情報記載用）
   - カスタマイズ可能な色・サイズ

5. **データ永続化**
   - ER図データとレイアウト情報の保存
   - Dockerボリュームによる柔軟な保存場所設定

### 技術スタック

- フロントエンド: TypeScript（ブラウザベース）
- バックエンド: Node.js 18以上、TypeScript 5.0以上
- データベース: MySQL
- デプロイ: Docker（Dockerイメージ: tkuni83/er-viewer）
- スキーマ管理: TypeSpec（main.tsp）からの型生成

### 現状

- プロトタイピング段階
- MVP（Minimum Viable Product）として機能している
- Dockerイメージとして配布中（docker run --rm --network host tkuni83/er-viewer）
- ポート30033で動作
- ホストOS上のMySQLへの接続をサポート

## リサーチ出力に求める事項

以下の観点について、連番を付けて調査・提案してください：

### 1. 既存の類似ツール・サービスの名称調査

既存のER図ビューア、ER図ツール、データベース可視化ツールの名称を調査し、名前の重複を避けるための参考情報を提示してください。

### 2. ER図ビューアであることが伝わる名称パターン ⚠ 意思決定が必要です

ER図ビューアであることが直感的に伝わる名称のパターンを提案してください。

**選択肢A**: ERやEntityRelationshipを含む直接的な名称
- 例: 「〇〇ER」「ER〇〇」「〇〇Entity」など
- メリット: 機能が一目で分かる
- デメリット: 汎用的すぎて差別化が難しい可能性

**選択肢B**: データベース/スキーマ/ダイアグラムを想起させる間接的な名称
- 例: Schema系、Diagram系、DB系の単語を含む
- メリット: ある程度の分かりやすさを保ちつつ独自性を出せる
- デメリット: ERに特化していることが伝わりにくい可能性

**選択肢C**: 造語や比喩的な名称
- 例: 全く新しい造語、または視覚化や構造を比喩する言葉
- メリット: 独自性が高く、ブランディングしやすい
- デメリット: 初見では何のツールか分かりにくい

### 3. 簡潔性と覚えやすさの観点 ⚠ 意思決定が必要です

1単語または2単語程度で、覚えやすく入力しやすい名称を提案してください。

**選択肢A**: 1単語の名称
- メリット: 最も簡潔で覚えやすい
- デメリット: 独自性を出すのが難しい

**選択肢B**: 2単語の名称
- メリット: 機能や特徴を表現しやすい
- デメリット: やや長くなる

### 4. 具体的な名称候補の提案

上記の調査結果を踏まえて、具体的な名称候補を10個以上提案してください。各候補について以下を記載してください：

- 名称
- 名称の由来・意味
- どの選択肢パターンに該当するか
- メリット
- デメリット
- 既存サービスとの重複チェック結果

推奨度の高い候補から順に記載してください。
