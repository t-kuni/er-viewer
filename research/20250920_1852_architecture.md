### 1. リポジトリ構成（例）

```
/app
 ├─ docker-compose.yml
 ├─ .env.example
 ├─ backend/
 │   ├─ src/
 │   ├─ Dockerfile.dev
 │   ├─ Dockerfile.prod
 │   └─ tsconfig.json
 ├─ frontend/
 │   ├─ public/
 │   ├─ src/
 │   ├─ Dockerfile.dev
 │   ├─ Dockerfile.prod
 │   └─ vite.config.ts
 └─ api-spec/
     └─ *.tsp   （TypeSpec 定義）
```

### 2. 開発環境

* **docker-compose.yml**

  * **backend**:

    * 画像: `node:20-alpine`
    * コマンド: `ts-node-dev --respawn src/index.ts`
    * ポート: `30033:30033`
    * 依存: `db`
  * **frontend**:

    * 画像: `node:20-alpine`
    * コマンド: `vite --host --port 5173`
    * ボリューム: `./frontend:/app`（ホットリロード）
  * **db**:

    * 画像: `mysql:8`
    * 環境変数: `MYSQL_ROOT_PASSWORD`, `MYSQL_DATABASE`
* `docker compose up` で3サービス同時起動。
* `frontend/public` 以下の変更で Vite が自動ビルドしブラウザを更新。
* MySQL は内部ネットワークのみ公開し、ホスト OS からの接続を許可しない。

### 3. バックエンド

* **言語／FW**: TypeScript + Express。
* **ポート**: `30033` 固定。
* **ORM**: Prisma または TypeORM。
* **API 仕様**: `api-spec/*.tsp` で定義し `tsp compile` → OpenAPI & TypeScript クライアント自動生成。
* **開発ビルド**: `ts-node-dev`。
* **本番ビルド**: multi-stage Dockerfile（`npm ci && npm run build && npm prune --production`）で軽量化。

### 4. フロントエンド

* **言語**: TypeScript（React + Vite 想定）。
* **API クライアント**: TypeSpec 生成コードを `src/api/` へ取り込み。
* **ホットリロード**: Vite 標準機能で `public` 変更を即時反映。
* **本番ビルド**: `npm run build` → 静的ファイル生成し Nginx などで配信。または同一 Node コンテナで `serve`.

### 5. 本番運用フロー

1. `docker build -f backend/Dockerfile.prod -t yourrepo/app-backend:tag .`
2. `docker build -f frontend/Dockerfile.prod -t yourrepo/app-frontend:tag .`
3. `docker push` で Docker Hub へ登録。
4. エンドユーザは

   ```bash
   docker run -d --name app \
     -e DB_URL="mysql://user:pass@host:3306/db" \
     -p 30033:30033 yourrepo/app-backend:tag
   ```

   フロントは別途 CDN 配信または `docker run`。
5. DB は含めず、環境変数で任意の MySQL/PostgreSQL 等に接続可能にする。

### 6. CI/CD（任意）

* GitHub Actions で `push` 時に lint/test → build → docker push。
* tag 付きリリースでプロダクション用イメージを発行。

上記で要件を満たすリアーキテクチャが実現できます。
